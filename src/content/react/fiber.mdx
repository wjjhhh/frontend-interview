## 核心概念

React Fiber 是 React 16 引入的新架构，将原来的递归不可中断的 Reconciliation 改造为**可中断的异步渲染**。

### 为什么需要 Fiber？

React 15 的 Stack Reconciler 使用递归遍历虚拟 DOM，一旦开始就无法中断，如果组件树很深，会长时间占用主线程，导致动画卡顿、用户交互无响应。

### Fiber 节点结构

每个 Fiber 节点是一个普通 JavaScript 对象：

```javascript
{
  tag: WorkTag,         // 组件类型
  key: null | string,
  type: any,            // 组件函数/类
  stateNode: any,       // DOM 节点或类实例
  return: Fiber | null, // 父 Fiber
  child: Fiber | null,  // 第一个子 Fiber
  sibling: Fiber | null,// 下一个兄弟 Fiber
  alternate: Fiber | null, // 双缓冲
  effectTag: SideEffectTag,
  memoizedState: any,   // Hooks 链表
}
```

## 高频面试题

### 1. Fiber 的工作流程

**两个阶段：**

1. **Render 阶段（可中断）**：遍历 Fiber 树，收集变更（Side Effects），不涉及 DOM 操作
2. **Commit 阶段（不可中断）**：将 Side Effects 同步应用到 DOM

```text
beginWork → 深度优先遍历子节点
completeWork → 回溯创建 DOM / 收集 Effects
commitWork → 同步提交到真实 DOM
```

### 2. 时间切片（Time Slicing）

Fiber 通过 `requestIdleCallback`（实际使用 `MessageChannel`）实现时间切片：

- 每个工作单元处理一个 Fiber 节点
- 处理完后检查是否还有剩余时间
- 如果时间不够，让出主线程，下一帧继续

### 3. 双缓冲机制

React 维护两棵 Fiber 树：
- **current 树**：当前屏幕上显示的内容
- **workInProgress 树**：正在构建的新树

构建完成后，通过指针切换将 workInProgress 变为 current，实现无闪烁更新。

## 深入追问

### React 的优先级调度

React 18 使用 Lane 模型管理优先级：
- **SyncLane**：同步更新（用户输入）
- **InputContinuousLane**：连续输入（滚动）
- **DefaultLane**：默认优先级
- **TransitionLane**：过渡更新（startTransition）
- **IdleLane**：空闲更新

### useTransition 和 useDeferredValue

```javascript
// useTransition：标记低优先级更新
const [isPending, startTransition] = useTransition();
startTransition(() => {
  setSearchResults(heavyComputation(input));
});

// useDeferredValue：延迟值更新
const deferredQuery = useDeferredValue(query);
```
