## 核心概念

闭包（Closure）是指一个函数能够记住并访问其词法作用域，即使该函数在其词法作用域之外执行。

闭包的形成需要三个条件：
- 函数嵌套
- 内部函数引用了外部函数的变量
- 内部函数在外部函数作用域之外被调用

## 高频面试题

### 1. 什么是闭包？请举例说明

**答案：**

闭包本质上是函数与其词法环境的组合。当一个函数被创建时，它会保持对其外部作用域变量的引用。

```javascript
function createCounter() {
  let count = 0;
  return {
    increment: () => ++count,
    getCount: () => count,
  };
}

const counter = createCounter();
counter.increment();
counter.increment();
console.log(counter.getCount()); // 2
```

### 2. 经典闭包陷阱：循环中的 var

```javascript
// 问题代码
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// 输出: 3, 3, 3

// 解决方案 1: 使用 let
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// 输出: 0, 1, 2

// 解决方案 2: 使用 IIFE
for (var i = 0; i < 3; i++) {
  ((j) => {
    setTimeout(() => console.log(j), 100);
  })(i);
}
```

### 3. 闭包的应用场景

- **数据私有化**：模块模式，隐藏内部状态
- **函数柯里化**：创建高阶函数
- **防抖/节流**：保持计时器引用
- **缓存/记忆化**：缓存计算结果

```javascript
// 记忆化函数
function memoize(fn) {
  const cache = new Map();
  return function (...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) return cache.get(key);
    const result = fn.apply(this, args);
    cache.set(key, result);
    return result;
  };
}
```

## 深入追问

### 闭包会导致内存泄漏吗？

闭包本身不会导致内存泄漏，但如果闭包持有对大量数据的引用且不被释放，就可能造成内存问题。解决方法：

- 及时解除不需要的引用（设为 `null`）
- 避免在闭包中引用不需要的外部变量
- 使用 `WeakMap`/`WeakRef` 存储对象引用

### V8 引擎如何优化闭包？

V8 通过「变量提升分析」只在闭包中保留实际被引用的变量，而非整个作用域。这种优化叫做 **Context Specialization**。

## 手写练习

> 实现一个 `once` 函数，使传入的函数只能被调用一次：

```javascript
function once(fn) {
  let called = false;
  let result;
  return function (...args) {
    if (called) return result;
    called = true;
    result = fn.apply(this, args);
    return result;
  };
}
```
