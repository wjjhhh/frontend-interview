## 核心概念

Vue 的响应式系统是其核心特性，通过数据劫持 + 发布订阅模式实现数据驱动视图更新。

### Vue 2 vs Vue 3

| 特性 | Vue 2 | Vue 3 |
|------|-------|-------|
| 核心 API | Object.defineProperty | Proxy |
| 数组监听 | 重写数组方法 | 原生支持 |
| 新增属性 | 需要 Vue.set | 自动响应 |
| 性能 | 递归遍历 | 懒代理 |

## 高频面试题

### 1. Vue 3 响应式核心实现

```javascript
// 简化版 reactive 实现
const targetMap = new WeakMap();
let activeEffect = null;

function reactive(target) {
  return new Proxy(target, {
    get(obj, key, receiver) {
      track(obj, key);
      const result = Reflect.get(obj, key, receiver);
      if (typeof result === 'object' && result !== null) {
        return reactive(result); // 懒代理
      }
      return result;
    },
    set(obj, key, value, receiver) {
      const result = Reflect.set(obj, key, value, receiver);
      trigger(obj, key);
      return result;
    },
  });
}

function track(target, key) {
  if (!activeEffect) return;
  let depsMap = targetMap.get(target);
  if (!depsMap) targetMap.set(target, (depsMap = new Map()));
  let deps = depsMap.get(key);
  if (!deps) depsMap.set(key, (deps = new Set()));
  deps.add(activeEffect);
}

function trigger(target, key) {
  const depsMap = targetMap.get(target);
  if (!depsMap) return;
  const deps = depsMap.get(key);
  if (deps) deps.forEach((effect) => effect());
}

function effect(fn) {
  activeEffect = fn;
  fn();
  activeEffect = null;
}
```

### 2. ref 和 reactive 的区别

- `ref`：基本类型包装为 `{ value: T }`，通过 `.value` 访问
- `reactive`：只接受对象类型，返回 Proxy 对象
- `ref` 在模板中自动解包，不需要 `.value`

### 3. computed 的实现原理

computed 本质上是一个带有 **lazy** 标志的 effect：
- 首次访问时计算并缓存结果
- 依赖变化时标记为 dirty
- 下次访问时重新计算

## 深入追问

### Vue 3 为什么选择 Proxy 替代 defineProperty？

1. Proxy 可以监听新增/删除属性
2. Proxy 可以监听数组下标变化
3. Proxy 是懒代理，性能更好（不用递归遍历）
4. Proxy 可以监听更多操作（has、deleteProperty 等）

### watchEffect vs watch

- `watchEffect`：自动收集依赖，立即执行
- `watch`：显式指定监听源，支持获取新旧值
